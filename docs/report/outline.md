# 리포트 구조 (Outline)

## 제목
**AI 개발 방법론 비교 분석: Ralph, BMAD, Superpowers, MOAI ADK**

---

## 구성 (8개 섹션)

### 1. Executive Summary (1p)
- 4개 방법론 한눈에 보는 요약표
- 핵심 차별점 3줄 요약

### 2. 조사 배경 및 목적 (0.5p)
- AI 보조 개발(AI-assisted development) 패러다임 부상 배경
- 4개 방법론 선정 이유 (GitHub 공개, 실제 사용 중, 다양한 접근법)
- 조사 방법: GitHub 레포 직접 분석 + 실제 설치 및 실습

### 3. 개별 방법론 심층 분석 (각 1~1.5p, 총 4~6p)
각 방법론에 대해 동일한 구조로 분석:
- 개요 및 핵심 철학
- 아키텍처 및 주요 기능
- 워크플로우/프로세스
- 강점 및 약점
- 실제 설치 및 실습 결과

#### 3-1. Ralph — 자율 AI 에이전트 루프
#### 3-2. BMAD Method — AI 팀 시뮬레이션 프레임워크
#### 3-3. Superpowers — AI 에이전트 스킬 주입 프레임워크
#### 3-4. MOAI ADK — 하네스 엔지니어링 기반 개발 킷 (한국 개발)

### 4. 통합 비교 분석 (1~1.5p)
- 다차원 비교표 (철학, 복잡도, 적용 범위, 비용 효율 등)
- 레이더 차트 (시각화)
- AI와 인간의 역할 분담 관점 비교

### 5. 직접 설치 및 실습 결과 (1p) ← 차별화 핵심
- 각 방법론 설치 과정 및 소감
- 실제 작동 확인 스크린샷
- 설치 중 발생한 이슈와 해결 방법

### 6. 방법론 선택 가이드 (0.5p)
- 상황별 추천 매트릭스
  - 혼자 MVP 빠르게 → Ralph
  - 체계적인 전체 기획 필요 → BMAD
  - 품질 중심 개발 프로세스 → Superpowers
  - Claude Code + 대규모 프로젝트 → MOAI ADK

### 7. 결론 및 개인 견해 (0.5p)
- AI 개발 방법론의 공통 트렌드 분석
- 개인적으로 가장 주목하는 방법론 + 이유
- 향후 AI 개발 방법론의 발전 방향 전망

### 8. 참고 자료
- 각 GitHub 레포지토리
- 공식 문서 링크
- 원작자 블로그/아티클
